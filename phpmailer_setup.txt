# PHPMailer Setup Instructions for Bluehost

## Option 1: Download PHPMailer Files (Recommended)

1. Download PHPMailer from: https://github.com/PHPMailer/PHPMailer/releases
2. Extract the files
3. Upload the following files to your server in a "PHPMailer" folder:

```
public_html/
├── sendemail.php
├── PHPMailer/
│   ├── src/
│   │   ├── PHPMailer.php
│   │   ├── SMTP.php
│   │   └── Exception.php
│   └── ...
```

## Option 2: Use Composer (if available)

If your Bluehost account supports Composer:

1. SSH into your account
2. Navigate to public_html directory
3. Run: `composer require phpmailer/phpmailer`

## Option 3: Manual File Creation

If you can't download, I can help you create the necessary PHPMailer files manually.

## File Structure After Setup

Your directory should look like this:

## What the Script Does

1. Connects to your database using the same connection as your admin panel
2. Checks these tables for new submissions (emailed = 0):
   - tour_booking
   - contact_submissions
   - build_submission
   - community_messages
3. Sends a formatted HTML email to fabrdaa@gmail.com with all new submissions
4. Marks processed records as emailed = 1
5. Logs all activity for monitoring

## Email Content

You'll receive emails with:
- Professional HTML formatting
- All new submissions organized by type
- Complete details from each database record
- Summary count of total new items
- Timestamp of when the report was generated

## Troubleshooting

If emails don't send:
1. Check email_automation_errors.log for error messages
2. Verify PHPMailer files are uploaded correctly
3. Confirm Gmail app password is correct
4. Test database connection
5. Check file permissions (644 for PHP files)

## Security Notes

- Your Gmail app password is stored in the PHP file
- The script only processes records marked as emailed = 0
- All database queries use proper escaping to prevent SQL injection
- Error logging helps with troubleshooting without exposing sensitive data
